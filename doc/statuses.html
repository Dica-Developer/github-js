Key/ value pair of request headers to pass along with the HTTP request. Valid headers are: If-Modified-Since, If-None-Match, Cookie, User-Agent

State of the status - can be one of pending, success, error, or failure. Validation rule: ^(pending|success|error|failure)$.

Target url to associate with this status. This URL will be linked from the GitHub UI to allow users to easily see the ‘source’ of the Status.
